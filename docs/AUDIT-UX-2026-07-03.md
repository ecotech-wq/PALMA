Statuts : A VERIFIER / CONFIRME / REJETE / CORRIGE (hash).

## 1. [P1-terrain] Suppression de compte sans confirmation modale — utilise confirm() JavaScript natif
- Source : parcours admin ; fichier : src/app/(app)/admin/users/UserActions.tsx
- Scenario : Admin clique sur le bouton Supprimer (poubelle) pour un utilisateur. Une popup JavaScript confirm() apparaît, mais sur téléphone c'est clair et l'utilisateur peut supprimer accidentellement en validant.
- Correction proposee : Remplacer le confirm() JavaScript par un DialogConfirm React/composant modale réutilisable avec boutons clairs (Annuler, Supprimer). La modale doit afficher le nom de l'utilisateur et ses conséquences.
- Statut : A VERIFIER

## 7. [P1-terrain] Rapport : 2 écrans au lieu de 1, perte de temps (terrain critique)
